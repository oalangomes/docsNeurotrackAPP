Integração com Agenda

O módulo de Integração com Agenda do NeurotrackApp permite o gerenciamento de compromissos com sincronização Google Calendar, ajudando pessoas neurodivergentes a organizar sua rotina de forma eficiente.

Diagrama de Fluxo

Principais Funcionalidades

Gerenciamento de Compromissos

  • Criação de eventos: Interface simplificada para adicionar compromissos
  • Categorização: Organização por tipo, prioridade e contexto
  • Lembretes personalizados: Notificações adaptadas ao perfil neurodivergente
  • Tempo de preparação: Configuração de alertas antecipados

Sincronização com Google Calendar

  • Integração bidirecional: Sincronização em tempo real
  • Filtros de sincronização: Controle sobre quais eventos são sincronizados
  • Resolução de conflitos: Gerenciamento inteligente de sobreposições

Visualização e Planejamento

  • Múltiplas visualizações: Diária, semanal, mensal e linha do tempo
  • Planejamento visual: Representação clara de duração e intervalos
  • Análise de carga: Avaliação de sobrecarga de compromissos

Endpoints da API

Método Endpoint Descrição
GET /agenda Listar eventos da agenda
POST /agenda Adicionar novo evento
GET /agenda/{id} Obter evento específico
PUT /agenda/{id} Atualizar evento
DELETE /agenda/{id} Excluir evento
POST /agenda/sync Sincronizar com Google Calendar
GET /agenda/conflicts Verificar conflitos de agenda
GET /agenda/availability Verificar disponibilidade de horários

Cenários de Uso

Criação de Compromisso com Preparação

  1. Usuário adiciona novo compromisso
  2. Define horário, local e detalhes
  3. Configura tempo necessário para preparação
  4. Sistema reserva tempo antes do evento para preparação
  5. Configura lembretes personalizados

Sincronização com Google Calendar

  1. Usuário configura integração com Google Calendar
  2. Define filtros de sincronização
  3. Sistema sincroniza eventos em ambas direções
  4. Notifica sobre possíveis conflitos

Planejamento Semanal

  1. Usuário acessa visualização semanal
  2. Sistema apresenta análise de carga de compromissos
  3. Sugere redistribuição em caso de sobrecarga
  4. Usuário ajusta eventos conforme necessário
